Linux CentOs 6下安装jdk和MySQL

发表于 2017-07-25 | 分类于 Linux | 评论数量 | 阅读次数
| 字数统计 1,257

今天在学习项目部署的时候看到文档中有关于。配置jdk和MySQL的介绍，虽然之前都是按照百度的，但是还是记录下来，以备以后使用。

安装JDK

查询并卸载JDK： 

[root@localhost ~]# java –version          //先查询系统的java版本     
    java version "1.6.0_18"       
    Java(TM) 2 Runtime Environment, Standard Edition (build 1.6.0_18-b02)        
    Java HotSpot(TM) Client VM (build 1.6.0_18-b02, mixed mode, sharing)            
    或者   

    java version "1.7.0_45"              
    OpenJDK Runtime Environment (rhel-2.4.3.3.el6-x86_64 u45-b15)               
    OpenJDK 64-Bit Server VM (build 24.45-b08, mixed mode)               

[root@localhost ~]#rpm -qa | grep jdk                
    java-1.6.0-openjdk-1.6.0.0-1.7.b09.el5

如有其他版本的JDK或者是OpenJDK，按照rpm -qa | grepjdk查询出来的相关包顺序从下往上逐个删除，使用以下命令 

[root@localhost ~]# yum -y remove java-1.6.0-openjdk-1.6.0.0-1.7.b09.el5     
    Setting up Remove Process          
    Resolving Dependencies......                
    Removed: java-1.6.0-openjdk-1.6.0.0-1.7.b09.el5                    
    ...... (开始卸载)                    
    Complete!(卸载完毕)                    
[root@localhost ~]# rpm -qa | grep gcj                  
    libgcj-4.1.2-48.el5                      
    java-1.4.2-gcj-compat-1.4.2.0-40jpp.115                      
[root@localhost ~]# yum -y remove java-1.4.2-gcj-compat-1.4.2.0-40jpp.115                     
[root@localhost ~]# yum -y remove libgcj-4.1.2-42.el5

这样卸载完成毕后，再查询系统的Java版本： 

[root@localhost ~]# java -version             
    bash: java: command not found

出现以上内容表明说明卸载成功!!!

上传JDK文件

将jdk-6u35-linux-i586-rpm.bin上传到服务器的根目录下
给jdk赋予读写的权限，命令为：chmod -R 755 jdk-6u35-linux-i586-rpm.bin
注：64位系统请安装64位相关软件包，复制命令时请确认版本号。 

安装JDK，命令为：                           
sh /jdk-6u35-linux-i586-rpm.bin                     
修改Java环境变量                 
vi /etc/profile

在该文件的最后添加以下三行代码：（按键盘上的【i】键，进入编辑模式） 

export JAVA_HOME=/usr/java/jdk1.6.0_35              
export PATH=$JAVA_HOME/bin:$PATH                    
export CLASSPATH=.:/usr/java/jdk1.6.0_35/lib/tools.jar:$CLASSPATH

添加之后，按键盘上的【ESC】键，在服务器上输入【:wq!】即可保存退出!!!
也可以直接修改服务器上该文件，保存即可。

测试安装是否完成

执行命令java –version，若出现下列内容则代表安装完成。 

java version "1.6.0_35"
Java(TM) SE Runtime Environment (build 1.6.0_35-b10)
Java HotSpot(TM) Server VM (build 20.10-b01, mixed mode)

安装数据库Mysql 5.6.26

查询并卸载其他版本： 

[root@localhost ~]# rpm -qa|grep -i mysql

如有其他版本的MySQL，按照查询出来的相关包顺序从下往上逐个删除，使用以下命令： 

[root@localhost ~]# rpm -e 查询出来的相关包    //rpm卸载MySQL

注意事项：
如果出现包依赖，使用 –nodeps参数强制删除。
（如：[root@localhost ~]# rpm -e MySQL-server-5.6.26-1.el6.x86_64 –nodeps）

上传Mysql相关RPM包

将MySQL-server-5.6.26-1.el6.x86_64.rpm和MySQL-client-5.6.26-1.el6.x86_64.rpm上传至/mnt/路径下 

[root@localhost ~]# cd /mnt/     //进入mnt目录                         
安装依赖环境：                  

yum install perl                  
yum install numactl                   
安装MYSQL：                      
[root@localhost mnt]# rpm -ivh MySQL-client-5.6.26-1.el6.x86_64.rpm               
[root@localhost mnt]# rpm -ivh MySQL-server-5.6.26-1.el6.x86_64.rpm               
如果报错：            

则执行：                
[root@localhost mnt]# rpm -ivh MySQL-client-5.6.26-1.el6.x86_64.rpm --nodeps             
[root@localhost mnt]# rpm -ivh MySQL-server-5.6.26-1.el6.x86_64.rpm --nodeps           
启动Mysql服务：            
[root@localhost ~]# service mysql start                   

[root@localhost ~]# netstat -anlt      //查看3306端口是否已经启动

以上表明Mysql服务已启动
查看Mysql版本： 

[root@localhost ~]# mysql -V

进入Mysql数据库
由于Mysql5.6.26安装时默认给root用户设置了随机密码，因此需要使用密码登陆数据库，密码存储在/root/.mysql_secret 文件中，打开文件最后一行显示的是最新的登陆密码 

[root@localhost ~]# cat /root/.mysql_secret                       

[root@localhost ~]# mysql -u root –p      //使用root登陆Mysql                       
Enter password:                         //输入密码                   

mysql>//在Mysql中执行以下命令             
mysql> set PASSWORD=PASSWORD(' 123456 ');                 
mysql> update mysql.user set password=password("XXXXXXX") where User="root";         
mysql> grant all PRIVILEGES on *.* to root@'%' identified by 'XXXXXXX';               
mysql> flush privileges;              
mysql> quit;

6、将所有上传的文件或文件夹都设置为777，或有可执行的权限；

rm -rf /root/idc
